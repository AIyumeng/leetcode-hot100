# 17. letter-combinations-of-a-phone-number

## 🔗 题目链接
[LeetCode 17. Letter Combinations Of A Phone Number](https://leetcode.com/problems/letter-combinations-of-a-phone-number/)

## 🧠 题目描述
给定一个仅包含数字 `2-9` 的字符串，返回所有它能表示的字母组合。答案可以按 **任意顺序** 返回。
数字到字母的映射（电话键盘）如下：
- 2 → "abc"
- 3 → "def"
- 4 → "ghi"
- 5 → "jkl"
- 6 → "mno"
- 7 → "pqrs"
- 8 → "tuv"
- 9 → "wxyz"

**示例**：
- 示例 1：输入：digits = "23"，输出：["ad","ae","af","bd","be","bf","cd","ce","cf"]；
- 示例 2：输入：digits = ""，输出：[]；
- 示例 3：输入：digits = "2"，输出：["a","b","c"]。

---

## 💡 解题思路
核心思路：
- 回溯（DFS）：将数字组合问题转化为「多叉树遍历」，每个数字对应一层节点，每个字母对应一个分支；
- 映射表辅助：用字典存储数字到字母的映射，快速获取当前数字对应的可选字母；
- 路径记录+回溯：递归过程中记录当前组合路径，遍历完所有数字时保存完整路径，回溯撤销当前选择。

步骤：
1. 处理边界：若输入为空字符串，直接返回空列表；
2. 构建数字-字母映射表，初始化结果列表和当前路径；
3. 递归回溯：
   - 终止条件：当前路径长度等于数字字符串长度，将路径加入结果列表；
   - 遍历当前数字对应的所有字母，将字母加入路径，递归处理下一个数字，回溯撤销当前字母；
4. 递归结束后返回结果列表。

DFS（深度优先搜索）是一种「先深后广」的遍历/搜索算法，核心是「递归+回溯」（也可通过栈实现非递归），适用于树、图、组合/排列等问题。下面给出 DFS 最通用的写法框架，并结合具体示例说明核心逻辑，新手可直接套用。

### 一、DFS 通用框架（递归版，最常用）
递归版 DFS 是最易理解、最常用的写法，核心包含 **「终止条件」「当前层处理」「递归下探」「回溯（可选）」** 四个部分：
```python
def dfs(参数):
    # 1. 终止条件：到达递归边界（如遍历完所有节点/满足目标条件）
    if 终止条件:
        记录结果/返回值
        return
    
    # 2. 遍历当前层的所有可能选择
    for 选择 in 当前层可选列表:
        # 3. 做出选择（标记/处理当前节点，可选：剪枝）
        处理当前选择（如加入路径、标记已访问）
        
        # 4. 递归下探：进入下一层
        dfs(更新后的参数)
        
        # 5. 回溯：撤销选择（恢复状态，供下一次循环使用）
        撤销当前选择（如移出路径、取消标记）
```

### 二、核心组件说明
| 组件         | 作用                                                                 | 示例场景                  |
|--------------|----------------------------------------------------------------------|---------------------------|
| 终止条件     | 防止递归无限循环，同时触发结果记录                                   | 树遍历到叶子节点、路径和达标 |
| 当前层遍历   | 枚举当前节点的所有可能分支                                           | 图的邻接节点、组合的可选元素 |
| 做出选择     | 标记当前节点的状态（避免重复访问/记录路径），可加「剪枝」优化         | 标记节点已访问、加入路径列表 |
| 递归下探     | 传递更新后的状态，进入下一层搜索                                     | 树的子节点、图的邻接节点   |
| 回溯         | 恢复状态，让当前层的下一个选择能基于干净的状态搜索（核心！）| 移出路径列表、取消节点标记 |

### 三、实战示例：组合总和（典型 DFS 应用）
以「LeetCode 39. 组合总和」为例，套用框架实现：
> 题目：给定无重复元素的数组 `candidates` 和目标值 `target`，找出所有和为 `target` 的组合（元素可重复选）。
```python
def combinationSum(candidates, target):
    res = []  # 存储最终结果
    
    def dfs(path, start, remain):
        # 1. 终止条件：剩余和为0（找到有效组合）或小于0（无效）
        if remain == 0:
            res.append(path.copy())  # 注意拷贝，避免后续修改影响结果
            return
        if remain < 0:
            return
        
        # 2. 遍历当前层可选元素（start 避免重复组合，如 [2,3] 和 [3,2]）
        for i in range(start, len(candidates)):
            num = candidates[i]
            # 3. 做出选择：将当前数加入路径
            path.append(num)
            
            # 4. 递归下探：剩余和减少，start=i 允许重复选当前数
            dfs(path, i, remain - num)
            
            # 5. 回溯：撤销选择
            path.pop()
    
    # 初始化调用：空路径、从第0个元素开始、剩余和为target
    dfs([], 0, target)
    return res

# 测试
print(combinationSum([2,3,6,7], 7))  # 输出 [[2,2,3], [7]]
```


### 四、关键技巧（避坑+优化）
1. **剪枝优化**：在「做出选择」前判断是否有必要继续（如组合总和中，若当前数 > 剩余和，直接跳过），减少无效递归；
2. **去重处理**：排序后跳过重复元素（如三数之和），或通过 `start` 控制遍历起点（如组合总和）；
3. **状态标记**：图的 DFS 需标记「已访问节点」（如 `visited` 数组/集合），避免环导致死循环；
4. **路径拷贝**：记录结果时需拷贝路径（如 `path.copy()`），否则回溯会修改已记录的结果。

### 总结
1. 递归版 DFS 核心框架：**终止条件 → 遍历选择 → 做出选择 → 递归下探 → 回溯**，是新手首选；
2. 回溯是 DFS 的灵魂：撤销选择才能让同一层的不同选择共享干净的状态；
3. 剪枝和去重是 DFS 优化的核心，能大幅降低时间复杂度。
