# 4. median-of-two-sorted-arrays

## 🔗 题目链接
[LeetCode 4. Median Of Two Sorted Arrays](https://leetcode.com/problems/median-of-two-sorted-arrays/)

## 🧠 题目描述
给定两个大小分别为 `m` 和 `n` 的**有序数组** `nums1` 和 `nums2`。请你找出并返回这两个有序数组的**中位数**。
要求算法的时间复杂度为 **O(log(m + n))**。

**示例**：
- 示例 1：输入：nums1 = [1,3], nums2 = [2]，输出：2.00000，解释：合并数组 = [1,2,3]，中位数是 2。
- 示例 2：输入：nums1 = [1,2], nums2 = [3,4]，输出：2.50000，解释：合并数组 = [1,2,3,4]，中位数是 (2 + 3) / 2 = 2.5。
- 示例 3：输入：nums1 = [0,0], nums2 = [0,0]，输出：0.00000。

---

## 💡 解题思路

核心思路：
- 二分查找（分治）：利用「中位数将数组划分为左右两部分，左半部分所有元素 ≤ 右半部分」的特性，在较短数组上做二分，将两个数组分别划分为左右两部分，满足「左半部分总长度 = 右半部分总长度（或多1）」且「左半部分最大值 ≤ 右半部分最小值」
- 缩短查找范围：始终在更短的数组上进行二分，将时间复杂度控制在 O(log(min(m,n)))（满足题目 O(log(m+n)) 要求）
- 边界处理：处理其中一个数组完全在左/右半部分的极端情况（如 nums1 所有元素都小于 nums2 最小值）

步骤：
1. 确保 `nums1` 是更短的数组（若不是则交换 nums1 和 nums2），减少二分次数；初始化二分边界 `left = 0`，`right = len(nums1)`
2. 二分查找划分点：
   - 计算 nums1 的划分点 `i = (left + right) // 2`，nums2 的划分点 `j = (m + n + 1) // 2 - i`（保证左半部分总长度 ≥ 右半部分，方便取左半最大值）
   - 处理边界值：nums1 左半最大值 `max_left1`（i=0 时为负无穷）、nums1 右半最小值 `min_right1`（i=m 时为正无穷）；同理计算 nums2 的 `max_left2` 和 `min_right2`
3. 调整划分点直至满足条件：
   - 若 `max_left1 > min_right2`：说明 nums1 左半部分过大，需将划分点左移（`right = i - 1`）
   - 若 `max_left2 > min_right1`：说明 nums1 左半部分过小，需将划分点右移（`left = i + 1`）
   - 若满足 `max_left1 ≤ min_right2` 且 `max_left2 ≤ min_right1`：找到正确划分点，计算中位数：
     - 若总长度为奇数：中位数 = max(max_left1, max_left2)
     - 若总长度为偶数：中位数 = (max(max_left1, max_left2) + min(min_right1, min_right2)) / 2

### 总结
1. 核心逻辑是通过二分查找找到两个数组的「正确划分点」，无需合并数组即可计算中位数，满足 O(log(min(m,n))) 的时间复杂度。
2. 划分点公式 `j = (m + n + 1) // 2 - i` 是关键：保证左半部分总长度比右半部分多 1（奇数长度时），让中位数直接等于左半最大值。
3. 边界值处理（负无穷/正无穷）是避免数组越界的核心技巧，需重点关注 i=0/i=m/j=0/j=n 的极端情况。
