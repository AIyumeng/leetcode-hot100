# 128. longest-consecutive-sequence

## 🔗 题目链接
[LeetCode 128. Longest Consecutive Sequence](https://leetcode.com/problems/longest-consecutive-sequence/)

## 🧠 题目描述
给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

**示例**：
- 示例 1：输入：nums = [100,4,200,1,3,2]，输出：4；
  解释：最长数字连续序列是 [1,2,3,4]，长度为4；
- 示例 2：输入：nums = [0,3,7,2,5,8,4,6,0,1]，输出：9；
  解释：最长数字连续序列是 [0,1,2,3,4,5,6,7,8]，长度为9；
- 示例 3：输入：nums = []，输出：0；
- 示例 4：输入：nums = [9,1,-3,2,4,8,3,-1,6,-2,-4,7]，输出：4（序列：-4,-3,-2,-1 或 6,7,8,9）。

---

## 💡 解题思路
核心思路：
- 哈希集合优化（最优）：将数组存入哈希集合，仅对「序列起点」（即 `num-1` 不在集合中）开始遍历，统计连续序列长度，避免重复计算；
- 排序+遍历（时间超界风险）：先排序数组，再遍历统计连续序列长度（时间复杂度O(n logn)，不满足题目O(n)要求）；
- 并查集（进阶思路）：将相邻数字合并到同一集合，最终统计最大集合的大小（时间复杂度O(n)，实现稍复杂）。

步骤（以哈希集合优化为例，最优解法）：
1. 边界处理：若数组为空，直接返回0；
2. 初始化：将数组转换为哈希集合 `num_set`（去重+O(1)查找），初始化最长长度 `max_len = 0`；
3. 遍历集合中的每个数字 `num`：
   - 判断是否为序列起点：若 `num - 1` 不在 `num_set` 中，说明是新序列的起点；
   - 统计当前序列长度：初始化 `current_num = num`，`current_len = 1`；
   - 循环查找 `current_num + 1` 是否在集合中，若存在则 `current_num += 1`，`current_len += 1`；
   - 更新最长长度：`max_len = max(max_len, current_len)`；
4. 遍历结束后返回 `max_len`。

### 总结
1. 核心逻辑：哈希集合的关键是「只从序列起点开始遍历」，避免对每个数字都统计连续长度（如数字2在1,2,3,4中，仅从1开始统计，2/3/4直接跳过），保证时间复杂度O(n)；
2. 关键细节：集合去重是必要的（如数组含重复数字时，不影响序列长度统计）；判断 `num-1` 是否存在是筛选起点的核心；
3. 复杂度：
   - 哈希集合法：时间O(n)（每个数字最多被访问两次：一次遍历，一次统计序列），空间O(n)（存储集合）；
   - 排序法：时间O(n logn)，空间O(1)（原地排序），不满足题目时间要求；
   - 并查集法：时间O(n)，空间O(n)，实现复杂度高于哈希集合；
4. 实战建议：面试中优先写哈希集合解法，逻辑简洁且满足O(n)时间要求；需注意空数组、重复数字、全负数等边界情况。
