# 33. search-in-rotated-sorted-array

## 🔗 题目链接
[LeetCode 33. Search In Rotated Sorted Array](https://leetcode.com/problems/search-in-rotated-sorted-array/)

## 🧠 题目描述
整数数组 `nums` 按升序排列，数组中的值 **互不相同**。
在传递给函数之前，`nums` 在预先未知的某个下标 `k`（`0 <= k < nums.length`）上进行了**旋转**，使数组变为 `[nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]`（下标 从 0 开始）。
例如， `[0,1,2,4,5,6,7]` 在下标 3 处旋转后可能变为 `[4,5,6,7,0,1,2]`。
给你 **旋转后** 的数组 `nums` 和一个整数 `target`，如果 `nums` 中存在这个目标值 `target`，则返回它的下标，否则返回 `-1`。

**要求**：必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

**示例**：
- 示例 1：输入：nums = [4,5,6,7,0,1,2], target = 0，输出：4；
- 示例 2：输入：nums = [4,5,6,7,0,1,2], target = 3，输出：-1；
- 示例 3：输入：nums = [1], target = 0，输出：-1。

---

## 💡 解题思路
核心思路：
- 二分查找变种：利用旋转数组的特性，每次二分后必有一半数组是有序的，缩小搜索范围；
- 有序区间判断：通过比较 `nums[mid]` 与 `nums[left]` 确定左/右半区是否有序；
- 目标值定位：在有序半区判断 target 是否在该区间内，调整左右指针缩小范围。

步骤：
1. 初始化左右指针 `left=0`、`right=len(nums)-1`；
2. 二分循环（`left <= right`）：
   - 计算中间索引 `mid = (left + right) // 2`，若 `nums[mid] == target`，直接返回 `mid`；
   - 判断左半区是否有序（`nums[left] <= nums[mid]`）：
     - 若有序且 `target` 在 `[nums[left], nums[mid])` 范围内，右指针移到 `mid-1`；
     - 否则左指针移到 `mid+1`；
   - 若右半区有序：
     - 若 `target` 在 `(nums[mid], nums[right]]` 范围内，左指针移到 `mid+1`；
     - 否则右指针移到 `mid-1`；
3. 循环结束未找到目标值，返回 `-1`。

### 总结
1. 核心逻辑是「二分查找+有序区间判断」：旋转数组虽整体无序，但二分后必有一半有序，利用这一特性实现 O(log n) 复杂度；
2. 有序区间判断关键：通过 `nums[left] <= nums[mid]` 区分左右半区是否有序，避免遗漏边界；
3. 目标值范围判断：仅在有序半区判断 target 是否存在，能快速缩小搜索范围，是解题的核心优化。
