# 70. climbing-stairs

## 🔗 题目链接
[LeetCode 70. Climbing Stairs](https://leetcode.com/problems/climbing-stairs/)

## 🧠 题目描述
假设你正在爬楼梯。需要 `n` 阶你才能到达楼顶。
每次你可以爬 `1` 或 `2` 个台阶。你有多少种不同的方法可以爬到楼顶呢？

**示例**：
- 示例 1：输入：n = 2，输出：2，解释：有两种方法可以爬到楼顶：1. 1 阶 + 1 阶；2. 2 阶；
- 示例 2：输入：n = 3，输出：3，解释：有三种方法可以爬到楼顶：1. 1 阶 + 1 阶 + 1 阶；2. 1 阶 + 2 阶；3. 2 阶 + 1 阶；
- 示例 3：输入：n = 4，输出：5。

---

## 💡 解题思路
核心思路：
- 动态规划（斐波那契数列）：到达第 `n` 阶的方法数 = 到达第 `n-1` 阶的方法数 + 到达第 `n-2` 阶的方法数（最后一步要么爬1阶，要么爬2阶）；
- 空间优化：无需存储完整DP数组，仅用两个变量滚动记录前两阶的方法数，空间复杂度从O(n)降至O(1)；
- 边界初始化：第1阶只有1种方法，第2阶有2种方法，作为动态规划的初始条件。

步骤：
1. 边界处理：若 `n <= 2`，直接返回 `n`（n=1返回1，n=2返回2）；
2. 初始化变量：`prev_prev = 1`（第1阶的方法数），`prev = 2`（第2阶的方法数）；
3. 遍历从3到n的阶数：
   - 计算当前阶数的方法数：`current = prev_prev + prev`；
   - 更新变量：`prev_prev = prev`，`prev = current`（滚动更新前两阶的方法数）；
4. 遍历结束后，`prev` 即为第 `n` 阶的方法数，返回该值。

### 总结
1. 核心逻辑：状态转移方程 `dp[n] = dp[n-1] + dp[n-2]` 本质是斐波那契数列，仅需调整初始值；
2. 空间优化关键：用两个变量替代数组，避免不必要的空间消耗，是该题的最优解法；
3. 复杂度：时间复杂度O(n)（一次遍历），空间复杂度O(1)（仅用常数变量）。
