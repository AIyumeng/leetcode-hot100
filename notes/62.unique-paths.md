# 62. unique-paths

## 🔗 题目链接
[LeetCode 62. Unique Paths](https://leetcode.com/problems/unique-paths/)

## 🧠 题目描述
一个机器人位于一个 `m x n` 网格的左上角 （起始点在下图中标记为 “Start” ）。
机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish”）。
问总共有多少条不同的路径？

**示例**：
- 示例 1：输入：m = 3, n = 7，输出：28；
- 示例 2：输入：m = 3, n = 2，输出：3，解释：从左上角开始，总共有 3 条路径可以到达右下角：
  1. 向右 → 向下 → 向下；
  2. 向下 → 向下 → 向右；
  3. 向下 → 向右 → 向下；
- 示例 3：输入：m = 7, n = 3，输出：28。

---

## 💡 解题思路
核心思路：
- 二维动态规划：构建 `m x n` 的二维DP数组，`dp[i][j]` 表示从左上角到达网格第 `i` 行第 `j` 列的路径数；
- 状态转移：到达 `(i,j)` 的路径只能来自上方 `(i-1,j)` 或左方 `(i,j-1)`，因此 `dp[i][j] = dp[i-1][j] + dp[i][j-1]`；
- 边界初始化：第一行所有位置路径数为1（只能向右走），第一列所有位置路径数为1（只能向下走）。

步骤：
1. 初始化 `m x n` 的二维DP数组 `dp`，所有元素初始化为0；
2. 边界初始化：
   - 遍历第一行（`i=0`，`j` 从0到n-1）：`dp[0][j] = 1`（只能从起点向右移动到达）；
   - 遍历第一列（`j=0`，`i` 从0到m-1）：`dp[i][0] = 1`（只能从起点向下移动到达）；
3. 填充DP数组剩余位置（`i` 从1到m-1，`j` 从1到n-1）：
   - 对每个位置 `(i,j)`，执行状态转移：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`；
4. 遍历完成后，`dp[m-1][n-1]` 即为到达右下角的总路径数，返回该值。

### 总结
1. 二维DP核心：通过二维数组完整记录每个位置的路径数，逻辑直观，是动态规划的基础写法；
2. 状态转移本质：每个位置的路径数是「上方+左方」路径数之和，符合“只能向下/向右移动”的规则；
3. 复杂度：时间复杂度O(mn)（遍历二维数组），空间复杂度O(mn)（存储二维DP数组），适合理解核心逻辑，后续可优化为一维数组。
