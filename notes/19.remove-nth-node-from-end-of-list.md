# 19. remove-nth-node-from-end-of-list

## 🔗 题目链接
[LeetCode 19. Remove Nth Node From End Of List](https://leetcode.com/problems/remove-nth-node-from-end-of-list/)

## 🧠 题目描述
给你一个链表，删除链表的倒数第 `n` 个结点，并且返回链表的头结点。
**进阶**：你能尝试使用一趟扫描实现吗？

**示例**：
- 示例 1：输入：head = [1,2,3,4,5], n = 2，输出：[1,2,3,5]；
- 示例 2：输入：head = [1], n = 1，输出：[]；
- 示例 3：输入：head = [1,2], n = 1，输出：[1]。

---

## 💡 解题思路
核心思路：
- 双指针（快慢指针）：用两个指针拉开 `n` 个节点的间距，一次遍历即可定位到倒数第 `n` 个节点；
- 虚拟头节点：避免删除头节点时的边界处理问题；
- 一次扫描优化：快慢指针同步移动，快指针到末尾时，慢指针恰好指向目标节点的前驱节点。

步骤：
1. 构建虚拟头节点，指向原链表头节点（简化头节点删除逻辑）；
2. 初始化快慢指针，均指向虚拟头节点，先让快指针向前移动 `n` 步；
3. 快慢指针同步向后移动，直到快指针到达链表末尾；
4. 慢指针的下一个节点即为倒数第 `n` 个节点，执行删除操作；
5. 返回虚拟头节点的下一个节点（新链表头）。

### 总结
1. 双指针是核心：快慢指针拉开 `n` 步间距，一次遍历即可定位目标节点，时间复杂度 O(L)（L 为链表长度），满足「一趟扫描」要求；
2. 虚拟头节点是关键：避免单独处理「删除头节点」的边界场景，简化代码逻辑；
3. 定位前驱节点：慢指针最终指向目标节点的前驱，直接修改 `next` 指针即可完成删除，无需额外遍历。
