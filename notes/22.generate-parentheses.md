# 22. generate-parentheses

## 🔗 题目链接
[LeetCode 22. Generate Parentheses](https://leetcode.com/problems/generate-parentheses/)

## 🧠 题目描述
数字 `n` 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且 **有效的** 括号组合。
有效括号组合需满足：左括号必须以正确的顺序闭合。

**示例**：
- 示例 1：输入：n = 3，输出：["((()))","(()())","(())()","()(())","()()()"]；
- 示例 2：输入：n = 1，输出：["()"]。

---

## 💡 解题思路
核心思路：
- 回溯（DFS）：通过递归遍历所有可能的括号组合，而非暴力枚举后验证（减少无效计算）；
- 剪枝约束：在递归过程中限制括号生成规则（左括号数 ≤ n、右括号数 ≤ 左括号数），确保生成的组合有效；
- 路径记录+回溯：记录当前括号组合路径，满足条件时保存结果，回溯撤销当前选择。

步骤：
1. 初始化结果列表，定义回溯函数（参数：当前组合、左括号数、右括号数）；
2. 回溯函数终止条件：左括号数和右括号数均等于 n，将当前组合加入结果列表；
3. 递归生成有效括号：
   - 若左括号数 < n，添加左括号，递归调用回溯函数（左括号数+1）；
   - 若右括号数 < 左括号数，添加右括号，递归调用回溯函数（右括号数+1）；
4. 调用回溯函数（初始组合为空、左/右括号数为0），返回结果列表。

### 总结
1. 核心优化点：「剪枝约束」替代「暴力枚举+验证」，仅生成有效组合，时间复杂度从 O(2^(2n)) 降至 O(4^n/√n)（卡特兰数复杂度）；
2. 回溯关键规则：左括号未达上限时可加左括号，右括号数小于左括号数时可加右括号，确保组合有效；
3. 回溯逻辑：通过参数传递当前组合（字符串拼接）实现隐式回溯，无需手动撤销选择，简洁高效。
