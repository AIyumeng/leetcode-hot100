# 64. minimum-path-sum

## 🔗 题目链接
[LeetCode 64. Minimum Path Sum](https://leetcode.com/problems/minimum-path-sum/)

## 🧠 题目描述
给定一个包含非负整数的 `m x n` 网格 `grid`，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

**说明**：每次只能向下或者向右移动一步。

**示例**：
- 示例 1：输入：grid = [[1,3,1],[1,5,1],[4,2,1]]，输出：7，解释：路径 1→3→1→1→1 的总和最小；
- 示例 2：输入：grid = [[1,2,3],[4,5,6]]，输出：12。

---

## 💡 解题思路
核心思路：
- 二维动态规划：构建 `m x n` 的DP数组，`dp[i][j]` 表示从左上角到达 `(i,j)` 的最小路径和；
- 状态转移：到达 `(i,j)` 的最小路径和 = 网格当前值 + min(上方路径和, 左方路径和)，即 `dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`；
- 原地优化：直接在原网格上更新，无需额外DP数组，将空间复杂度从O(mn)降至O(1)。

步骤：
1. 边界初始化（原地修改grid）：
   - 第一行：从第二个元素开始，每个位置的值 = 当前值 + 左侧值（只能从左方到达）；
   - 第一列：从第二个元素开始，每个位置的值 = 当前值 + 上方值（只能从上方到达）；
2. 填充网格剩余位置（`i` 从1到m-1，`j` 从1到n-1）：
   - 对每个位置 `(i,j)`，更新值为 `grid[i][j] + min(grid[i-1][j], grid[i][j-1])`；
3. 遍历完成后，`grid[m-1][n-1]` 即为从左上角到右下角的最小路径和，返回该值。

### 总结
1. 核心逻辑：状态转移方程是解题关键，通过比较上方/左方的最小路径和，确保每一步都选择最优路径；
2. 原地优化技巧：直接复用原网格存储DP状态，无需额外空间，是空间效率的核心优化；
3. 复杂度：时间复杂度O(mn)（遍历网格），空间复杂度O(1)（原地修改），是该问题的最优解。
