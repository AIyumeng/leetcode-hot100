# 104. maximum-depth-of-binary-tree

## 🔗 题目链接
[LeetCode 104. Maximum Depth Of Binary Tree](https://leetcode.com/problems/maximum-depth-of-binary-tree/)

## 🧠 题目描述
给定一个二叉树，找出其最大深度。

二叉树的深度为根节点到最远叶子节点的最长路径上的节点数。

**说明**：叶子节点是指没有子节点的节点。

**示例**：
- 示例 1：输入：root = [3,9,20,null,null,15,7]，输出：3；
  解释：
  ```
      3
     / \
    9  20
      /  \
     15   7
  ```
  根节点3到叶子节点15/7的路径长度为3（3→20→15 或 3→20→7），是最长路径；
- 示例 2：输入：root = [1,null,2]，输出：2；
- 示例 3：输入：root = []，输出：0；
- 示例 4：输入：root = [0]，输出：1。

---

## 💡 解题思路
核心思路：
- 递归法（后序遍历）：树的最大深度 = 1 + 左子树最大深度 和 右子树最大深度的较大值，递归遍历左右子树并返回深度；
- 迭代法（BFS）：利用层序遍历的特性，每遍历完一层深度加1，最终层数即为最大深度；
- 迭代法（DFS）：用栈模拟递归，记录每个节点的深度，遍历过程中更新最大深度。

步骤（以递归法为例，最优解法）：
1. 终止条件：若根节点为空，返回深度0；
2. 递归计算左子树的最大深度 `left_depth = maxDepth(root.left)`；
3. 递归计算右子树的最大深度 `right_depth = maxDepth(root.right)`；
4. 当前节点的深度为 `1 + max(left_depth, right_depth)`，返回该值。

### 总结
1. 核心逻辑：递归法贴合「树的深度 = 1 + 子树最大深度」的定义，代码极简；BFS法通过层数直接对应深度，逻辑直观；
2. 关键细节：递归终止条件是「空节点返回0」，确保叶子节点的深度计算为1（1 + 0）；
3. 复杂度：
   - 递归法：时间O(n)（遍历所有节点），空间O(h)（h为树的高度，最坏为n，平衡树为logn）；
   - BFS法：时间O(n)，空间O(n)（队列最多存储一层节点）；
   - DFS法：时间O(n)，空间O(h)（栈存储节点和深度）；
4. 实战建议：面试中优先写递归法，代码简洁且易理解；若要求非递归，BFS法更不易出错。
