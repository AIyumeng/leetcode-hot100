# 55. jump-game

## 🔗 题目链接
[LeetCode 55. Jump Game](https://leetcode.com/problems/jump-game/)

## 🧠 题目描述
给定一个非负整数数组 `nums`，你最初位于数组的**第一个下标**。数组中的每个元素代表你在该位置可以跳跃的最大长度。

判断你是否能够到达最后一个下标。

**示例**：
- 示例 1：输入：nums = [2,3,1,1,4]，输出：true，解释：可以先跳 1 步，从下标 0 到 1，然后再跳 3 步到达最后一个下标；
- 示例 2：输入：nums = [3,2,1,0,4]，输出：false，解释：无论如何，总会到达下标 3，该位置的跳跃长度为 0，无法继续前进，因此无法到达最后一个下标。

---

## 💡 解题思路
核心思路：
- 贪心策略（反向验证）：从最后一个位置倒推，寻找能到达终点的最近位置，逐步向前验证是否能回到起点；
- 正向覆盖（最优）：遍历数组时维护「当前能到达的最远位置」，若遍历到的位置超出最远可达范围则返回false，否则更新最远可达范围；
- 边界判断：若最远可达范围覆盖最后一个下标，直接返回true，无需遍历到底。

步骤：
1. 初始化「最远可达位置」`max_reach = 0`，数组长度 `n = len(nums)`；
2. 遍历数组每个下标 `i`：
   - 若当前下标 `i` 超过 `max_reach`（无法到达当前位置），返回false；
   - 更新 `max_reach` 为 `max(max_reach, i + nums[i])`（当前位置能跳到的最远位置）；
   - 若 `max_reach >= n - 1`（已能到达最后一个下标），提前返回true；
3. 遍历结束后，判断 `max_reach >= n - 1`，返回对应结果。

### 总结
1. 核心逻辑：贪心策略的核心是「只关注最远可达范围」，无需记录具体跳跃路径，大幅简化逻辑；
2. 效率优化：提前终止条件（`max_reach >= n-1`）可减少不必要的遍历，时间复杂度为O(n)；
3. 边界处理：重点判断「当前下标是否在可达范围内」，这是区分能否继续前进的关键。
