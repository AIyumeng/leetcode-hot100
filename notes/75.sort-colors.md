# 75. sort-colors

## 🔗 题目链接
[LeetCode 75. Sort Colors](https://leetcode.com/problems/sort-colors/)

## 🧠 题目描述
给定一个包含红色、白色和蓝色、共 `n` 个元素的数组 `nums`，**原地**对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色顺序排列。

我们使用整数 `0`、`1` 和 `2` 分别表示红色、白色和蓝色。

**要求**：必须在不使用库的排序函数的情况下解决这个问题，且尽量减少操作次数。

**示例**：
- 示例 1：输入：nums = [2,0,2,1,1,0]，输出：[0,0,1,1,2,2]；
- 示例 2：输入：nums = [2,0,1]，输出：[0,1,2]；
- 示例 3：输入：nums = [0]，输出：[0]。

---

## 💡 解题思路
核心思路：
- 三指针（荷兰国旗算法）：用三个指针划分区间，分别指向「0的右边界」「当前遍历位置」「2的左边界」；
- 一次遍历排序：遍历过程中根据当前元素值（0/1/2）调整指针位置，将元素归位到对应区间；
- 原地操作：无需额外空间，仅通过交换元素完成排序，时间复杂度O(n)。

步骤：
1. 初始化指针：`left = 0`（0的右边界，初始在数组最左端）、`curr = 0`（当前遍历指针）、`right = len(nums) - 1`（2的左边界，初始在数组最右端）；
2. 遍历数组（`curr <= right`）：
   - 若 `nums[curr] == 0`：交换 `nums[curr]` 和 `nums[left]`，`left` 右移（0的区间扩大），`curr` 右移；
   - 若 `nums[curr] == 1`：无需交换，`curr` 直接右移；
   - 若 `nums[curr] == 2`：交换 `nums[curr]` 和 `nums[right]`，`right` 左移（2的区间扩大），`curr` 不移动（交换过来的元素需重新判断）；
3. 遍历结束后，数组已按0、1、2有序排列。

### 总结
1. 核心逻辑：三指针通过一次遍历完成分区，避免了计数排序的两次遍历，是该题的最优解法；
2. 指针意义：`left` 左侧全为0，`right` 右侧全为2，`[left, curr)` 之间全为1，遍历过程中维护这个区间定义；
3. 关键细节：交换2时 `curr` 不移动，因为交换过来的元素可能是0或2，需要重新判断；交换0时 `curr` 可移动，因为交换过来的是1（`left` 到 `curr` 之间已遍历过，都是1）。
