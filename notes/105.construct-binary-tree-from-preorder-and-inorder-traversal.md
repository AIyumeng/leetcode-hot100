# 105. construct-binary-tree-from-preorder-and-inorder-traversal

## 🔗 题目链接
[LeetCode 105. Construct Binary Tree From Preorder And Inorder Traversal](https://leetcode.com/problems/construct-binary-tree-from-preorder-and-inorder-traversal/)

## 🧠 题目描述
给定两个整数数组 `preorder` 和 `inorder`，其中 `preorder` 是二叉树的**先序遍历**，`inorder` 是同一棵树的**中序遍历**，请构造二叉树并返回其根节点。

**说明**：
- 树中没有重复的元素；
- `1 <= preorder.length <= 3000`；
- `inorder.length == preorder.length`。

**示例**：
- 示例 1：输入：preorder = [3,9,20,15,7]，inorder = [9,3,15,20,7]，输出：[3,9,20,null,null,15,7]；
  解释：构造的二叉树结构为：
  ```
      3
     / \
    9  20
      /  \
     15   7
  ```
- 示例 2：输入：preorder = [-1]，inorder = [-1]，输出：[-1]。

---

## 💡 解题思路
核心思路：
- 先序遍历特性：第一个元素是根节点；中序遍历特性：根节点左侧是左子树节点，右侧是右子树节点；
- 递归构建：通过先序找到根节点，在中序中定位根节点，拆分出左右子树的先序/中序数组，递归构建左右子树；
- 哈希表优化：用哈希表存储中序数组「值-索引」映射，避免每次查找根节点索引的O(n)时间。

步骤：
1. 边界处理：若先序/中序数组为空，返回空节点；
2. 初始化哈希表：存储inorder中每个值对应的索引（仅初始化一次，优化查找效率）；
3. 递归构建函数（参数：先序左边界pl、先序右边界pr，中序左边界il、中序右边界ir）：
   - 取preorder[pl]作为当前根节点值；
   - 在inorder中找到根节点索引`root_idx`，计算左子树节点数`left_size = root_idx - il`；
   - 构建根节点，递归构建左子树（pl+1到pl+left_size，il到root_idx-1）；
   - 递归构建右子树（pl+left_size+1到pr，root_idx+1到ir）；
   - 返回当前根节点；
4. 调用递归函数（初始边界：pl=0、pr=len(preorder)-1，il=0、ir=len(inorder)-1），返回根节点。

### 总结
1. 核心逻辑：利用先序「根左右」、中序「左根右」的特性拆分左右子树，递归构建是核心；
2. 关键优化：哈希表将中序根节点查找时间从O(n)降至O(1)，整体时间复杂度从O(n²)优化为O(n)；
3. 复杂度：
   - 时间O(n)（遍历所有节点+哈希表O(1)查找）；
   - 空间O(n)（哈希表存储索引+递归栈深度，最坏为n，平衡树为logn）；
4. 实战建议：面试中需重点说明「如何拆分左右子树的边界」，这是该题的易错点；需注意树中无重复元素是哈希表可用的前提。
