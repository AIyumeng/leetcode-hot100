# 102. binary-tree-level-order-traversal

## 🔗 题目链接
[LeetCode 102. Binary Tree Level Order Traversal](https://leetcode.com/problems/binary-tree-level-order-traversal/)

## 🧠 题目描述
给你二叉树的根节点 `root`，返回其节点值的**层序遍历**结果（即逐层地，从左到右访问所有节点）。

**示例**：
- 示例 1：输入：root = [3,9,20,null,null,15,7]，输出：[[3],[9,20],[15,7]]；
  解释：
  ```
      3
     / \
    9  20
      /  \
     15   7
  ```
  第一层：[3]，第二层：[9,20]，第三层：[15,7]；
- 示例 2：输入：root = [1]，输出：[[1]]；
- 示例 3：输入：root = []，输出：[]。

---

## 💡 解题思路
核心思路：
- 广度优先搜索（BFS）：用队列存储当前层的所有节点，遍历完当前层后，再处理下一层，天然适配层序遍历的「逐层访问」需求；
- 分层记录：每次遍历前记录当前队列的长度（即当前层的节点数），确保只处理当前层的节点，避免不同层节点混合；
- 空节点过滤：遍历过程中跳过空节点，仅将非空的子节点加入队列，保证结果的整洁性。

步骤：
1. 初始化：若根节点为空，直接返回空列表；创建队列并加入根节点，创建结果列表 `res`；
2. 循环处理队列（队列非空时）：
   - 记录当前层节点数 `level_size = len(queue)`，创建当前层结果列表 `level_res`；
   - 遍历当前层的 `level_size` 个节点：
     - 弹出队列头部节点，将节点值加入 `level_res`；
     - 若节点有左子节点，将左子节点加入队列；
     - 若节点有右子节点，将右子节点加入队列；
   - 将 `level_res` 加入结果列表 `res`；
3. 循环结束后，返回 `res`。

### 总结
1. 核心逻辑：BFS通过队列「先进先出」的特性实现逐层遍历，`level_size` 是分层的关键（避免跨层处理）；
2. 关键细节：必须先记录当前层节点数，再遍历，否则队列长度会随子节点加入动态变化，无法准确分层；
3. 复杂度：
   - BFS法：时间O(n)（遍历所有节点），空间O(n)（队列最多存储一层节点，最坏为n/2）；
   - 递归法：时间O(n)，空间O(n)（递归栈深度，最坏为树的高度）。
4. 实战建议：面试中优先写BFS解法，逻辑更贴合层序遍历的直观理解，且不易出错。
