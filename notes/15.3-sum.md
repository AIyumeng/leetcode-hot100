# 15. 3-sum

## 🔗 题目链接
[LeetCode 15. 3 Sum](https://leetcode.com/problems/3-sum/)

## 🧠 题目描述
给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k`，同时还满足 `nums[i] + nums[j] + nums[k] == 0`。请你返回所有和为 `0` 且不重复的三元组。
**注意**：答案中不可以包含重复的三元组。

**示例**：
- 示例 1：输入：nums = [-1,0,1,2,-1,-4]，输出：[[-1,-1,2],[-1,0,1]]；
- 示例 2：输入：nums = [0,1,1]，输出：[]；
- 示例 3：输入：nums = [0,0,0]，输出：[[0,0,0]]。

---

## 💡 解题思路
核心思路：
- 排序去重：先对数组排序，既方便双指针查找，也能快速跳过重复元素避免结果重复；
- 固定单值+双指针：将三数之和转化为「固定一个数 + 两数之和」问题，用双指针降低时间复杂度；
- 贪心收缩指针：根据当前三数之和与0的大小关系，移动左右指针，快速逼近目标值。

步骤：
1. 排序数组，初始化结果列表；
2. 遍历数组，固定第一个数 `nums[i]`：
   - 若 `nums[i] > 0`，直接终止遍历（排序后后续数更大，和不可能为0）；
   - 跳过重复的 `nums[i]`（避免重复三元组）；
3. 对每个 `nums[i]`，用左指针 `left = i+1`、右指针 `right = len(nums)-1` 查找另外两个数：
   - 计算三数之和 `sum = nums[i] + nums[left] + nums[right]`；
   - 若 `sum == 0`：记录三元组，同时跳过左右指针的重复值，收缩指针继续查找；
   - 若 `sum < 0`：左指针右移（增大和）；
   - 若 `sum > 0`：右指针左移（减小和）；
4. 遍历结束后返回结果列表。

### 总结
1. 核心优化点：排序 + 双指针将时间复杂度从暴力枚举的 O(n³) 降至 O(n²)，是该问题的最优解；
2. 去重是关键：需分别对「固定的第一个数」「左指针」「右指针」做重复值跳过，避免结果重复；
3. 提前终止：当固定数 `nums[i] > 0` 时直接终止遍历，减少无效计算。
