# 124. binary-tree-maximum-path-sum

## 🔗 题目链接
[LeetCode 124. Binary Tree Maximum Path Sum](https://leetcode.com/problems/binary-tree-maximum-path-sum/)

## 🧠 题目描述
路径 被定义为一条从树中任意节点出发，沿父节点-子节点连接，达到任意节点的序列。同一个节点在一条路径序列中**至多出现一次**。该路径**至少包含一个**节点，且不一定经过根节点。

路径和 是路径中各节点值的总和。

给你一个二叉树的根节点 `root`，返回其**最大路径和**。

**示例**：
- 示例 1：输入：root = [1,2,3]，输出：6；
  解释：最优路径是 2 → 1 → 3 ，路径和为 2+1+3 = 6；
- 示例 2：输入：root = [-10,9,20,null,null,15,7]，输出：42；
  解释：最优路径是 15 → 20 → 7 ，路径和为 15+20+7 = 42；
- 示例 3：输入：root = [2,-1]，输出：2；
- 示例 4：输入：root = [-3]，输出：-3。

---

## 💡 解题思路
核心思路：
- 后序遍历（递归）：遍历每个节点时，计算「以该节点为顶点的最大路径和」（左子树贡献 + 节点值 + 右子树贡献），同时返回「该节点能向上传递的最大路径和」（节点值 + 左/右子树的最大贡献，取非负）；
- 全局最大值维护：用全局变量/类变量记录遍历过程中所有节点的最大路径和，最终返回该值；
- 负贡献过滤：子树的贡献若为负，则舍弃（取0），避免拉低整体路径和。

步骤：
1. 初始化全局变量 `max_sum` 为负无穷（处理全负数的情况）；
2. 定义递归函数（参数：当前节点 `node`）：
   - 终止条件：若节点为空，返回0；
   - 递归计算左子树的最大贡献 `left_gain = max(dfs(node.left), 0)`（负贡献取0）；
   - 递归计算右子树的最大贡献 `right_gain = max(dfs(node.right), 0)`；
   - 计算当前节点为顶点的路径和 `current_sum = node.val + left_gain + right_gain`；
   - 更新全局最大路径和 `max_sum = max(max_sum, current_sum)`；
   - 返回当前节点能向上传递的最大贡献 `node.val + max(left_gain, right_gain)`；
3. 调用递归函数（参数为根节点）；
4. 返回全局变量 `max_sum`。

### 总结
1. 核心逻辑：后序遍历的核心是「先算子树贡献，再算当前节点路径和」，区分「顶点路径和」（用于更新全局最大值）和「向上传递贡献」（用于父节点计算）；
2. 关键细节：过滤负贡献（取max(贡献, 0)）是避免路径和被拉低的关键，即使全节点为负，也能正确返回最大的单个节点值；
3. 复杂度：
   - 时间O(n)（遍历所有节点）；
   - 空间O(h)（h为树的高度，递归栈深度，最坏为n，平衡树为logn）；
4. 实战建议：面试中需重点说明「顶点路径和」与「向上传递贡献」的区别，以及负贡献过滤的原因；全局变量可改用列表/类属性（Python中避免嵌套函数的变量作用域问题）。
