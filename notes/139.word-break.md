# 139. word-break

## 🔗 题目链接
[LeetCode 139. Word Break](https://leetcode.com/problems/word-break/)

## 🧠 题目描述
给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。请你判断是否可以利用字典中出现的单词拼接出 `s`（不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用）。

**注意**：
- 字典中的单词没有重复；
- 1 <= s.length <= 300；
- 1 <= wordDict.length <= 1000；
- 1 <= wordDict[i].length <= 20。

**示例**：
- 示例 1：输入：s = "leetcode"，wordDict = ["leet", "code"]，输出：true；
  解释："leetcode" 可以由 "leet" 和 "code" 拼接而成；
- 示例 2：输入：s = "applepenapple"，wordDict = ["apple", "pen"]，输出：true；
  解释："applepenapple" = "apple" + "pen" + "apple"，字典单词可重复使用；
- 示例 3：输入：s = "catsandog"，wordDict = ["cats", "dog", "sand", "and", "cat"]，输出：false；
- 示例 4：输入：s = "aaaaaaa"，wordDict = ["aaaa","aa"]，输出：true（如 "aaaa"+"aaa" 或 "aa"+"aa"+"aaa" 等）。

---

## 💡 解题思路
核心思路：
- 动态规划（DP）：定义 `dp[i]` 表示字符串 `s` 的前 `i` 个字符（即 `s[0:i]`）是否可以被字典拼接，通过状态转移判断每个位置的可行性；
- 记忆化递归：递归判断子串是否可拼接，用哈希表/数组记录已计算的子问题结果，避免重复计算；
- 暴力回溯（低效）：枚举所有可能的单词分割点，递归验证（时间复杂度指数级，会超时）。

步骤（以动态规划为例，最优解法）：
1. 预处理：将 `wordDict` 转换为哈希集合（`word_set`），方便O(1)查找；
2. 初始化DP数组：`dp[0] = true`（空字符串可被拼接），`dp[1...len(s)]` 初始为 `false`；
3. 遍历字符串每个位置 `i`（1到len(s)）：
   - 遍历分割点 `j`（0到i-1）：
     - 若 `dp[j] == true` 且 `s[j:i]` 在 `word_set` 中，则 `dp[i] = true`，跳出内层循环；
4. 遍历结束后返回 `dp[len(s)]`。

### 总结
1. 核心逻辑：动态规划的关键是「状态定义（前i个字符是否可拼接）+ 状态转移（分割点j验证子串）」，哈希集合优化单词查找效率；
2. 关键细节：`dp[0] = true` 是边界条件（空串合法），内层循环找到一个合法分割点即可跳出，无需继续验证；
3. 复杂度：
   - 动态规划法：时间O(n²)（n为s长度，i遍历n次，j遍历i次），空间O(n)（DP数组+哈希集合）；
   - 记忆化递归：时间O(n²)，空间O(n)（递归栈+记忆数组）；
   - 暴力回溯：时间O(2ⁿ)，空间O(n)（递归栈），会超时；
4. 实战建议：面试中优先写动态规划解法，逻辑清晰且效率可接受；可优化内层循环范围（如仅遍历字典中单词的长度，减少j的遍历次数）。
