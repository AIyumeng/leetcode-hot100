# 121. best-time-to-buy-and-sell-stock

## 🔗 题目链接
[LeetCode 121. Best Time To Buy And Sell Stock](https://leetcode.com/problems/best-time-to-buy-and-sell-stock/)

## 🧠 题目描述
给定一个数组 `prices`，它的第 `i` 个元素 `prices[i]` 表示一支给定股票第 `i` 天的价格。

你只能选择**某一天**买入这只股票，并选择在**未来的某一个不同的日子**卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 `0`。

**示例**：
- 示例 1：输入：prices = [7,1,5,3,6,4]，输出：5；
  解释：在第 2 天（价格 = 1）买入，在第 5 天（价格 = 6）卖出，最大利润 = 6-1 = 5。注意利润不能是 7-1 = 6，因为卖出价格需要大于买入价格；
- 示例 2：输入：prices = [7,6,4,3,1]，输出：0；
  解释：在这种情况下，没有交易完成，所以最大利润为 0；
- 示例 3：输入：prices = [1,2]，输出：1。

---

## 💡 解题思路
核心思路：
- 一次遍历（贪心）：遍历过程中记录「当前最低买入价」，并计算当前价格与最低买入价的差值（即当前利润），更新最大利润；

步骤：
1. 边界处理：若价格数组长度小于2，直接返回0；
2. 初始化：最低买入价 `min_price = prices[0]`，最大利润 `max_profit = 0`；
3. 遍历价格数组（从第2天开始）：
   - 更新最低买入价：`min_price = min(min_price, prices[i])`；
   - 计算当前利润：`current_profit = prices[i] - min_price`；
   - 更新最大利润：`max_profit = max(max_profit, current_profit)`；
4. 遍历结束后返回 `max_profit`。

### 总结
1. 核心逻辑：一次遍历法的关键是「动态维护最低买入价」，确保每次计算的利润都是当前最优解，时间复杂度最优；
2. 关键细节：必须先更新最低买入价，再计算当前利润，避免「先卖后买」的逻辑错误；
3. 复杂度：
   - 一次遍历法：时间O(n)，空间O(1)（仅用常量变量）；
4. 实战建议：面试中优先写一次遍历法，逻辑简洁且效率最高；若要求用动态规划，需掌握状态转移方程的推导。



# 122. best-time-to-buy-and-sell-stock-ii

## 🔗 题目链接
[LeetCode 122. Best Time To Buy And Sell Stock Ii](https://leetcode.com/problems/best-time-to-buy-and-sell-stock-ii/)

## 🧠 题目描述
给你一个整数数组 `prices`，其中 `prices[i]` 表示某支股票第 `i` 天的价格。

在每一天，你可以决定是否购买和/或出售股票。你在任何时候**最多**只能持有一股股票。你也可以先购买，然后在**同一天**出售。

返回你能获得的**最大利润**。

**示例**：
- 示例 1：输入：prices = [7,1,5,3,6,4]，输出：7；
  解释：在第 2 天（价格 = 1）买入，在第 3 天（价格 = 5）卖出，利润 = 5-1 = 4；
        接着在第 4 天（价格 = 3）买入，在第 5 天（价格 = 6）卖出，利润 = 6-3 = 3；
        总利润为 4+3 = 7；
- 示例 2：输入：prices = [1,2,3,4,5]，输出：4；
  解释：在第 1 天买入，第 5 天卖出，利润 = 5-1 = 4；（或每天买卖：(2-1)+(3-2)+(4-3)+(5-4)=4）；
- 示例 3：输入：prices = [7,6,4,3,1]，输出：0；
  解释：无交易可做，最大利润为 0。

---

## 💡 解题思路
核心思路：
- 贪心算法（累加正差价）：只要后一天价格高于前一天，就在前一天买入、后一天卖出，累加所有正差价即为最大利润（无需考虑长期持有，短期正收益累加等价于最大利润）；
- 动态规划：用 `dp[i][0]` 表示第i天不持有股票的最大利润，`dp[i][1]` 表示第i天持有股票的最大利润，通过状态转移求解；
- 暴力回溯（基础思路，效率低）：枚举所有买卖组合，计算利润并取最大值（时间复杂度高，仅适合小数据量）。

步骤（以贪心算法为例，最优解法）：
1. 边界处理：若价格数组长度小于2，直接返回0；
2. 初始化最大利润 `max_profit = 0`；
3. 遍历价格数组（从第2天开始）：
   - 计算当天与前一天的差价 `diff = prices[i] - prices[i-1]`；
   - 若差价为正，将差价累加到 `max_profit`；
4. 遍历结束后返回 `max_profit`。

### 总结
1. 核心逻辑：贪心算法的关键是「赚所有能赚的正差价」，因为多次短期交易的利润和等于一次长期持有（如1→5→6的利润= (5-1)+(6-5)=6-1）；
2. 关键细节：允许当天买卖，因此无需考虑持仓状态，只需关注相邻天数的差价；
3. 复杂度：
   - 贪心算法：时间O(n)，空间O(1)（仅用常量变量）；
   - 动态规划法：时间O(n)，空间O(n)（可优化为O(1)）；
4. 实战建议：面试中优先写贪心算法，逻辑极简且效率最高；若要求用动态规划，需掌握状态转移方程（`dp[i][0] = max(dp[i-1][0], dp[i-1][1]+prices[i])`；`dp[i][1] = max(dp[i-1][1], dp[i-1][0]-prices[i])`）。




# 123. best-time-to-buy-and-sell-stock-iii

## 🔗 题目链接
[LeetCode 123. Best Time To Buy And Sell Stock Iii](https://leetcode.com/problems/best-time-to-buy-and-sell-stock-iii/)

## 🧠 题目描述
给定一个数组 `prices`，它的第 `i` 个元素 `prices[i]` 表示一支给定的股票在第 `i` 天的价格。

设计一个算法来计算你所能获取的最大利润。你最多可以完成**两笔**交易。

**注意**：你不能同时参与多笔交易（你必须在再次购买前出售掉之前的股票）。

**示例**：
- 示例 1：输入：prices = [3,3,5,0,0,3,1,4]，输出：6；
  解释：在第 4 天（价格 = 0）买入，在第 6 天（价格 = 3）卖出，利润 = 3-0 = 3；
        接着在第 7 天（价格 = 1）买入，在第 8 天（价格 = 4）卖出，利润 = 4-1 = 3；
        总利润为 3+3 = 6；
- 示例 2：输入：prices = [1,2,3,4,5]，输出：4；
  解释：在第 1 天买入，第 5 天卖出，完成一笔交易，利润 = 4（最多两笔但一笔已最优）；
- 示例 3：输入：prices = [7,6,4,3,1]，输出：0；
  解释：无交易可做，最大利润为 0；
- 示例 4：输入：prices = [1,2,4,2,5,7,2,4,9,0]，输出：13；
  解释：第一笔：1买入→7卖出（利润6），第二笔：2买入→9卖出（利润7），总利润13。

---

## 💡 解题思路
核心思路：
- 动态规划（状态机）：定义4个关键状态，分别对应「第一次持有」「第一次卖出」「第二次持有」「第二次卖出」，通过状态转移计算每个状态的最大利润；
- 前后拆分法：先计算「第i天前完成一次交易的最大利润」，再计算「第i天后完成一次交易的最大利润」，两者相加的最大值即为最多两笔交易的最大利润；
- 暴力枚举（低效）：枚举两次交易的买入卖出点，计算利润和（时间复杂度O(n³)，仅理论可行）。

步骤（以动态规划状态机为例，最优解法）：
1. 边界处理：若价格数组长度小于2，直接返回0；
2. 初始化状态：
   - `buy1`：第一次买入的最大利润（初始为 `-prices[0]`，表示买入第一天股票）；
   - `sell1`：第一次卖出的最大利润（初始为 0，未交易）；
   - `buy2`：第二次买入的最大利润（初始为 `-prices[0]`，买入第一天股票，后续可卖出再买入）；
   - `sell2`：第二次卖出的最大利润（初始为 0，未交易）；
3. 遍历价格数组（从第2天开始）：
   - 更新 `buy1 = max(buy1, -prices[i])`（不操作 或 买入当天股票）；
   - 更新 `sell1 = max(sell1, buy1 + prices[i])`（不操作 或 卖出第一次持有的股票）；
   - 更新 `buy2 = max(buy2, sell1 - prices[i])`（不操作 或 第一次卖出后买入当天股票）；
   - 更新 `sell2 = max(sell2, buy2 + prices[i])`（不操作 或 卖出第二次持有的股票）；
4. 遍历结束后返回 `max(sell1, sell2)`（最多两笔，取一笔/两笔的最大值）。

### 总结
1. 核心逻辑：动态规划状态机的关键是拆分「两次交易的4个核心状态」，确保状态转移覆盖所有合法操作（买入前必须卖出）；
2. 关键细节：初始化时`buy1/buy2`为`-prices[0]`，表示第一天买入的成本，`sell1/sell2`初始为0表示无交易；
3. 复杂度：
   - 动态规划法：时间O(n)，空间O(1)（仅用4个状态变量）；
   - 前后拆分法：时间O(n)，空间O(n)（需存储前后两次交易的利润数组）；
4. 实战建议：面试中优先写动态规划状态机解法，空间复杂度最优且逻辑清晰；需注意「最多两笔交易」，最终结果需取`sell1`和`sell2`的最大值（可能一笔交易更优）。
