# 2. add-two-numbers

## 🔗 题目链接
[LeetCode 2. Add Two Numbers](https://leetcode.com/problems/add-two-numbers/)

## 🧠 题目描述
给你两个**非空**的链表，表示两个非负的整数。它们每位数字都是按照**逆序**的方式存储的，并且每个节点只能存储**一位**数字。
请你将两个数相加，并以相同形式返回一个表示和的链表。
你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

**示例**：
输入：l1 = [2,4,3], l2 = [5,6,4]
输出：[7,0,8]
解释：342 + 465 = 807，由于链表是逆序存储，因此结果链表为 [7,0,8]。

---

## 💡 解题思路

核心思路：
- 模拟手工逐位加法逻辑：利用链表逆序存储的特性，从链表头部（对应数字个位）开始相加，通过变量记录进位
- 哑节点简化链表操作：用 dummy 节点作为结果链表的虚拟头，避免单独处理头节点为空的边界情况
- 全覆盖循环条件：以「l1 未遍历完 或 l2 未遍历完 或 仍有进位」作为循环结束条件，确保所有位和进位都被处理

步骤：
1. 初始化哑节点 `dummy` 作为结果链表的虚拟头，`cur` 指针指向 `dummy`，进位变量 `c` 初始化为 0
2. 进入循环（条件：l1 或 l2 不为空，或进位 c 不为 0）：
   - 取出 l1 当前节点值 `a`（l1 为空则取 0），取出 l2 当前节点值 `b`（l2 为空则取 0）
   - 计算当前位总和 `d = a + b + c`，更新进位 `c = d // 10`，计算当前位结果 `e = d % 10`
   - 创建值为 `e` 的新节点，挂载到 `cur.next`，并将 `cur` 指针移动到新节点
   - 移动 l1 指针（l1 不为空则指向 next，否则保持不变），移动 l2 指针（逻辑同 l1）
3. 循环结束后，返回 `dummy.next`（结果链表的真实头节点）

### 总结
1. 核心逻辑匹配代码变量命名（如 `c` 表示进位、`a/b` 表示两链表当前值、`d` 表示总和、`e` 表示当前位结果），完全贴合代码执行流程。
2. 循环条件覆盖「l1/l2 未遍历完 + 进位未清零」，是处理「999+1=1000」这类边界场景的关键。
3. 链表指针移动采用 `l1 = l1.next if l1 else l1` 的简洁写法，确保空链表不会触发异常。
