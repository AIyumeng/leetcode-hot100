# 72. edit-distance

## 🔗 题目链接
[LeetCode 72. Edit Distance](https://leetcode.com/problems/edit-distance/)

## 🧠 题目描述
给你两个单词 `word1` 和 `word2`，请计算将 `word1` 转换成 `word2` 所使用的最少操作数。

你可以对一个单词进行如下三种操作：
1. 插入一个字符；
2. 删除一个字符；
3. 替换一个字符。

**示例**：
- 示例 1：输入：word1 = "horse", word2 = "ros"，输出：3；
  解释：
  horse → rorse (替换 'h' 为 'r')
  rorse → rose (删除 'r')
  rose → ros (删除 'e')
- 示例 2：输入：word1 = "intention", word2 = "execution"，输出：5；
  解释：
  intention → inention (删除 't')
  inention → enention (替换 'i' 为 'e')
  enention → exention (替换 'n' 为 'x')
  exention → exection (替换 'n' 为 'c')
  exection → execution (插入 'u')

## 💡 解题思路
核心思路：
- 二维动态规划：构建 `(m+1) x (n+1)` 的DP数组（m、n为两个单词长度），`dp[i][j]` 表示将 `word1` 前 `i` 个字符转换为 `word2` 前 `j` 个字符的最少操作数；
- 状态转移：根据字符是否相等分情况，优先选择插入/删除/替换中操作数最少的方式；
- 边界初始化：其中一个单词为空时，操作数等于另一个单词的长度（全插入/全删除）。

步骤：
1. 初始化：设 `m = len(word1)`，`n = len(word2)`，构建 `(m+1) x (n+1)` 的DP数组 `dp`；
2. 边界赋值：
   - 第一行（`i=0`）：`dp[0][j] = j`（空字符串转 `word2` 前j个字符，需插入j次）；
   - 第一列（`j=0`）：`dp[i][0] = i`（`word1` 前i个字符转空字符串，需删除i次）；
3. 填充DP数组（`i` 从1到m，`j` 从1到n）：
   - 若 `word1[i-1] == word2[j-1]`（当前字符相等）：`dp[i][j] = dp[i-1][j-1]`（无需操作）；
   - 若字符不相等：`dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])`（1为操作数，min内分别对应删除、插入、替换）；
4. 遍历完成后，`dp[m][n]` 即为最少编辑距离，返回该值。

### 总结
1. 核心逻辑：DP数组的状态转移覆盖了所有可能的编辑操作，通过比较三种操作的最小代价保证结果最优；
2. 边界意义：空字符串与非空字符串的转换是基础场景，为后续状态转移提供初始值；
3. 复杂度：时间复杂度O(mn)（遍历二维数组），空间复杂度O(mn)（存储DP数组）
